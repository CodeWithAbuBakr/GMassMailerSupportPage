GMassMailer – Chrome Extension

Email Campaign Manager for Gmail

Overview

GMassMailer is a Chrome extension that automates sending personalized Gmail campaigns. Users can upload a CSV file of contacts, configure sending intervals, and set daily limits. It ensures a seamless experience while complying with Gmail's policies.

Main Features

  • Gmail OAuth2 Authentication with token refresh
  • Upload & validate CSV with up to 10,000 contacts
  • Custom intervals, max emails per day, and daily start time
  • Live campaign tracking with progress bar and countdown
  • Auto CSV download with sent timestamps on stop or completion
  • Client-side data storage using chrome.storage.local
  • Centralized error handling and safe messaging

Getting Started

  1. Open the GMassMailer extension popup.
  2. Authenticate your Gmail using the Authenticate button.
  3. Upload a CSV file with these columns: First name, Last name, Email, Subject, Body.
  4. Configure:
    • Interval between emails
    • Max emails to send per day
    • Daily start time
  5. Click Start Sending and monitor progress.
  6. Click Stop anytime to halt the campaign and download the CSV.
Tip: GMassMailer automatically resumes the campaign each day based on your settings!

CSV Guidelines

CSV files must be under 5MB and contain up to 10,000 valid rows. The extension auto-detects headers and validates email formats. Invalid rows are skipped.

On campaign completion, a timestamped CSV is downloaded with a new column: Sent Date.

Authentication Details

Uses chrome.identity.getAuthToken({ interactive: true }) to initiate OAuth2 login. Access tokens are stored securely. If expired, the token refreshes automatically, or the user is prompted to re-authenticate.

Campaign Tracking

The popup displays real-time stats like sent count, progress, next email time, and a countdown to the next reset. The UI refreshes every 3 seconds to keep you updated.

Storage & State

All user data and state are stored locally via chrome.storage.local including:

  • accessToken, contacts, interval, maxPerDay
  • sentToday, isRunning, campaignCompleted
  • lastSentTime, nextEmailTime, and more

Error Handling

All errors—authentication, invalid CSVs, network issues—are caught and mapped to user-friendly messages via a centralized handler. Technical logs are sent to the console for debugging.

Security & Privacy

  • OAuth2-based Gmail access (Chrome Identity API)
  • All data stored locally; no external servers
  • User input/output is sanitized to prevent XSS/code injection

Developer Notes

  • popup.js handles UI, state, validation
  • background.js manages email logic (via messaging)
  • Use window.testCompletion for testing
  • Modular codebase, easy to extend
